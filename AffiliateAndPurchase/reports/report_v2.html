What Changed

v2 → v2.1: UA region classification fix

❌ INVALIDATED (v2): UA classified as CIS/EPN

v2 included Ukraine (UA) in CIS_COUNTRIES, causing UA users to be analyzed under the CIS proxy-return framework. CLAUDE.md explicitly states: "UA must be treated as Global / Portals, not CIS / EPN." UA is now analyzed with direct sk logic.

❌ INVALIDATED (v2): "UA anomaly — 3.9% proxy return despite 73.9% hub reach"

This was a methodological artifact. UA users were being evaluated with the CIS proxy signal (aliexpress.ru visit within 120s), which is the wrong metric for Global/Portals traffic. Under correct Global sk logic, UA shows 71.2% sk-return — completely normal for a European country.

❌ INVALIDATED (v2): "stock-tops.com and discount-tops.net show 24–25% CIS proxy return"

This apparent anomaly was caused by UA users (who route via Global portals, not aliexpress.ru) being counted in the CIS hub proxy-return metric. With UA correctly classified as Global, CIS hub proxy return rates are: market-tops 66.5%, tools-tops 66.2%, discount-tops 60.3%, stock-tops 64.2% — no anomaly.

✅ FIXED: UA is Global/Portals, analyzed via sk logic

UA: 3,175 eligible users, 2,347 hub reach (73.9%), 2,261 sk-return (71.2%) [GLOBAL_DIRECT]. This matches the expected range for European Global countries (ES: 75.8%, PL: 77.2%, FR: 77.0%).

✅ FIXED: UA PC events now use Global reason codes

~13,840 UA Purchase Completed events moved from CIS to Global classification. Now correctly analyzed with NO_OUR_SK_IN_72H, AF_AFTER_OUR_SK etc. [GLOBAL_DIRECT].

v1 → v2: CIS/EPN limited-observability framework

❌ INVALIDATED (v1): CIS hub sk-return comparison

v1 compared hub performance using sk-return rate for ALL users including CIS. CIS traffic uses EPN affiliate return (utm_*), not sk. The observed low sk-return for CIS users was a methodological artifact.

❌ INVALIDATED (v1): CIS reason codes in Problem B

v1 applied Global sk-based codes (NO_OUR_SK_IN_72H, FOREIGN_SK_AFTER_OUR_SK) to CIS events. EPN attribution is NOT_OBSERVABLE from historical events.

✅ REPLACED (v2): CIS uses proxy-return signal + CIS-specific reason codes

CIS proxy return = aliexpress.ru event within 120s of Affiliate Click [CIS_PROXY]. Problem B CIS: CIS_LIKELY_DELAYED_POSTBACK / CIS_PURCHASE_COMPLETED_WITHOUT_PURCHASE_UNDER_LIMITED_OBSERVABILITY / CIS_NO_HUB_REACH_OBSERVED.

Data Quality & Observability Caveats

CIS EPN Attribution Not Observable NOT_OBSERVABLE

MongoDB events do not store utm_source / utm_medium / utm_campaign. AliHelper EPN return pattern (utm_source=aerkol, utm_campaign=*_7685) cannot be reconstructed from historical data.

CIS Proxy Return Is Indirect CIS_PROXY

aliexpress.ru event within 120s of Affiliate Click proves site return only, not EPN param preservation.

Affiliate Click Cache Coverage for Problem B

Cache covers Mar 6 – Apr 3. PC events from Feb 27 – Mar 8 have incomplete 72h AC lookback. CIS_NO_HUB_REACH_OBSERVED may be overstated for that ~10-day period.

Cashback Observability Partial

cashback_list in PC is client-reported. Cashback site visits not logged to backend. Actual interference rate likely higher than observed 1.6%.

No order_id in Most Purchase Completed

Primary matching is user + time proximity (±10 min). Ambiguous matches treated as matched.

guestStateHistory = Config Delivery, Not Usage

Records confirm config was delivered. They do NOT confirm a redirect was executed using that config.

No Direct Auto-Redirect Log

webNavigation.onBeforeNavigate is not logged to backend. Auto-redirect opportunity is reconstructed indirectly.

Only _id Index on events and guestStateHistory

All queries use _id-based date range. No field-level index on guest_id in events. Aggregation scans the full window.

Recommendations

Quick Wins (days)

  • Fix Edge auto-redirect: investigate why Firefox reaches hub at 89.7% vs Edge at 75.2%. Check webNavigation timing in Edge.
  • Instrument Affiliate Click tracking: 396 users have our sk in events but no Affiliate Click — Mixpanel event not firing.
  • Investigate Global A6 (1,996 users reached hub, no sk return): check hub redirect URL injection logic.

Medium-Term Product Changes

  • Improve DOGI activation: 99.5% of the activation gap is users with working config who never interact with DOGI. A/B test coin placement, alternative triggers for Chrome.
  • Investigate AF overwrite (58,381 Global events, 24.4% of unmatched): identify which affiliate programs set last_af and negotiate attribution rules.
  • Homepage activation improvement: only 6.6% of homepage-only users reach hub.

Tracking / Instrumentation (critical)

  • [CRITICAL for CIS] Store utm_source, utm_medium, utm_campaign in MongoDB events for aliexpress.ru return visits. Without this, 45,047 CIS PC events per 28-day window have no observable attribution state.
  • Add Affiliate Return Detected client-side Mixpanel event for both Global sk returns and CIS EPN utm_* returns.
  • Persist last_epn_campaign, last_epn_source, last_epn_medium, last_epn_datetime analogous to last_sk for Global.
  • Add server-side cashback visit logging.

Follow-Up Analyses

  • CIS per-PC proxy return: targeted MongoDB query to compute aliexpress.ru proxy return per individual CIS PC event for Problem B classification.
  • Extend Affiliate Click cache to cover Feb 24 – Mar 6 to remove coverage gap in CIS 72h lookback for Problem B.
  • Global A6 investigation: why do 1,996 users reach hub but return without our sk?
