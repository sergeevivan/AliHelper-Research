Что изменилось

v2 → v2.1: исправление классификации региона UA

❌ АННУЛИРОВАНО (v2): UA отнесена к CIS/EPN

В v2 Украина (UA) была включена в CIS_COUNTRIES, и UA-пользователи анализировались по CIS-методологии косвенного возврата. CLAUDE.md явно указывает: «UA must be treated as Global / Portals, not CIS / EPN». Теперь UA анализируется через прямую логику sk.

❌ АННУЛИРОВАНО (v2): «Аномалия UA — 3.9% proxy return при 73.9% hub reach»

Это был методологический артефакт. UA-пользователи оценивались по CIS-прокси-сигналу (визит aliexpress.ru в течение 120с), что неверно для Global/Portals-трафика. По правильной Global sk-логике UA показывает 71.2% sk-return — совершенно нормально для европейской страны.

❌ АННУЛИРОВАНО (v2): «stock-tops.com и discount-tops.net показывают 24–25% CIS proxy return»

Эта кажущаяся аномалия была вызвана тем, что UA-пользователи (которые маршрутизируются через Global порталы, а не aliexpress.ru) учитывались в CIS-метрике proxy return для хабов. После корректной классификации UA как Global CIS-метрики proxy return: market-tops 66.5%, tools-tops 66.2%, discount-tops 60.3%, stock-tops 64.2% — аномалии нет.

✅ ИСПРАВЛЕНО: UA является Global/Portals, анализируется через логику sk

UA: 3 175 eligible пользователей, 2 347 достигли хаба (73.9%), 2 261 вернулись с sk (71.2%) [GLOBAL_DIRECT]. Соответствует ожидаемому диапазону для европейских Global-стран (ES: 75.8%, PL: 77.2%, FR: 77.0%).

✅ ИСПРАВЛЕНО: PC-события UA теперь используют коды причин Global

~13 840 событий Purchase Completed из UA переведены из CIS в Global-классификацию. Теперь корректно анализируются с NO_OUR_SK_IN_72H, AF_AFTER_OUR_SK и т.д. [GLOBAL_DIRECT].

v1 → v2: фреймворк ограниченной наблюдаемости CIS/EPN

❌ АННУЛИРОВАНО (v1): сравнение sk-return хабов для CIS

v1 сравнивал производительность хабов через sk-return rate ДЛЯ ВСЕХ пользователей, включая CIS. CIS-трафик использует EPN-возврат (utm_*), а не sk. Наблюдаемый низкий sk-return для CIS-пользователей был методологическим артефактом.

❌ АННУЛИРОВАНО (v1): коды причин CIS в Проблеме B

v1 применял Global sk-коды (NO_OUR_SK_IN_72H, FOREIGN_SK_AFTER_OUR_SK) к CIS-событиям. EPN-атрибуция NOT_OBSERVABLE из исторических events.

✅ ЗАМЕНЕНО (v2): CIS использует proxy-return сигнал + CIS-специфичные коды причин

CIS proxy return = событие aliexpress.ru в течение 120с после Affiliate Click [CIS_PROXY]. Проблема B для CIS: CIS_LIKELY_DELAYED_POSTBACK / CIS_PURCHASE_COMPLETED_WITHOUT_PURCHASE_UNDER_LIMITED_OBSERVABILITY / CIS_NO_HUB_REACH_OBSERVED.

Ограничения данных и наблюдаемости

CIS EPN-атрибуция не наблюдаема NOT_OBSERVABLE

MongoDB events не хранят utm_source / utm_medium / utm_campaign. Паттерн AliHelper EPN-возврата (utm_source=aerkol, utm_campaign=*_7685) не может быть восстановлен из исторических данных.

CIS Proxy Return — косвенный сигнал CIS_PROXY

Событие aliexpress.ru в течение 120с после Affiliate Click доказывает лишь возврат на сайт, но не сохранность EPN-параметров.

Покрытие кеша Affiliate Click для Проблемы B

Кеш покрывает 6 марта – 3 апреля. PC-события с 27 февраля по 8 марта имеют неполное 72-часовое покрытие AC. CIS_NO_HUB_REACH_OBSERVED может быть завышен за этот ~10-дневный период.

Кешбэк-наблюдаемость частичная

cashback_list в PC — данные клиентской стороны. Визиты на кешбэк-сайты не логируются на бэкенде. Реальный уровень вмешательства, вероятно, выше наблюдаемых 1.6%.

order_id отсутствует в большинстве Purchase Completed

Основное сопоставление — по пользователю + временной близости (±10 мин). Неоднозначные совпадения считаются совпавшими.

guestStateHistory = доставка конфига, не использование

Записи подтверждают доставку конфига. Они НЕ подтверждают факт выполнения редиректа с этим конфигом.

Нет прямого лога авто-редиректа

webNavigation.onBeforeNavigate не логируется на бэкенде. Возможность авто-редиректа восстанавливается косвенно.

Только индекс _id на events и guestStateHistory

Все запросы используют диапазон дат через _id. Нет полевого индекса по guest_id в events. Агрегация сканирует всё окно.

Рекомендации

Быстрые победы (дни)

  • Исправить авто-редирект Edge: расследовать, почему Firefox достигает хаба в 89.7% vs Edge 75.2%. Проверить тайминг webNavigation в Edge.
  • Инструментировать трекинг Affiliate Click: у 396 пользователей есть наш sk в events, но нет Affiliate Click — событие Mixpanel не отправляется.
  • Расследовать Global A6 (1 996 польз. достигли хаба, нет sk-возврата): проверить логику инъекции sk в URL редиректа хаба.

Среднесрочные продуктовые изменения

  • Улучшить активацию DOGI: 99.5% разрыва активации — пользователи с рабочим конфигом, которые никогда не взаимодействуют с DOGI. A/B-тест размещения монетки, альтернативные триггеры для Chrome.
  • Расследовать перезапись AF (58 381 Global-событий, 24.4% несовпавших): определить, какие affiliate-программы устанавливают last_af, и согласовать правила атрибуции.
  • Улучшить активацию на главной странице: только 6.6% пользователей только с главной страницей достигают хаба.

Трекинг / Инструментирование (критично)

  • [КРИТИЧНО для CIS] Хранить utm_source, utm_medium, utm_campaign в MongoDB events для возвратов на aliexpress.ru. Без этого 45 047 CIS PC-событий за 28-дневное окно не имеют наблюдаемого состояния атрибуции.
  • Добавить клиентское событие Mixpanel Affiliate Return Detected для возвратов как с Global sk, так и с CIS EPN utm_*.
  • Сохранять last_epn_campaign, last_epn_source, last_epn_medium, last_epn_datetime аналогично last_sk для Global.
  • Добавить серверное логирование визитов на кешбэк-сайты.

Дополнительные анализы

  • CIS proxy return на PC-событие: целевой MongoDB-запрос для вычисления proxy-возврата aliexpress.ru на каждое CIS PC-событие для классификации в Проблеме B.
  • Расширить кеш Affiliate Click до покрытия 24 февр. – 6 марта для устранения разрыва в 72-часовом CIS-lookback для Проблемы B.
  • Расследование Global A6: почему 1 996 пользователей достигают хаба, но возвращаются без нашего sk?

Дорожная карта инструментирования

Без хранения utm_* для CIS, 45 047 CIS PC-событий за 28-дневное окно — это чёрный ящик атрибуции. Это единственный наиболее приоритетный пробел в инструментировании.
